PR #874 — Changes Made Today (April 25, 2026)
================================================

These changes address PR review comments from ErikMogensen and fix additional bugs found during testing.


1. PR REVIEW FIXES
-------------------
- Guard loadQueues/loadTopics with !IsEventHubNamespace in MainForm.ShowEntities
- Use QueueExists("$__scope_probe__$") for scope validation instead of enumerating all queues — single lightweight REST call that returns false on SB and throws UnauthorizedAccessException on EH namespaces
- Match status codes "401"/"40100" instead of string "Unauthorized" for more robust error detection
- Capture eventHubMessagingFactory in local variable to fix TOCTOU race condition


2. RECONNECT STATE LEAK FIX
-----------------------------
- Reset IsEventHubNamespace = false and eventHubMessagingFactory = null at start of Connect() to prevent stale state when switching namespaces
- Gate Event Hub loading behind confirmed IsEventHubNamespace + AAD connection in MainForm.ShowEntities
- Updated ConnectFormAadTests to expect EventHubEntities in AAD entity list


3. ENTITYPATH BUG FIX
----------------------
When EntityPath is set (e.g., "def"), GetQueues/GetTopics threw MessagingException because the entity is an event hub, not a queue/topic. Then all event hubs got loaded instead of just the specified one.

- Try-catch MessagingException in GetQueueUsingEntityPath / GetTopicUsingEntityPath — returns empty list instead of throwing (ServiceBusQueue.cs, ServiceBusTopic.cs)
- Load only the specified event hub via GetEventHubAsync(entityPath) instead of GetEventHubsAsync() when EntityPath is set (MainForm.cs)
- Expose EntityPath property on ServiceBusHelper
